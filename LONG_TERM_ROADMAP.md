# 长线能力路线

## 多人联机

建议先做合作模式，不做对战：

1. 把 `game` 状态拆成可同步的纯数据快照。
2. 把输入拆成 `input commands`，例如移动、射击、购买。
3. 使用 WebSocket 服务同步：
   - 房主或服务器负责怪物 AI、掉落、伤害判定。
   - 客户端只上传输入并渲染状态。
4. 先支持 2 人合作，再扩展到 4 人。

风险点：

- 当前怪物和掉落逻辑都在本地即时计算，直接联机会出现不同步。
- 需要先把随机数改成可复现种子，或者全部由服务器生成。

## 关卡编辑器

建议先做离线编辑器：

1. 独立新页面 `editor.html`。
2. 编辑内容：
   - 森林装饰物
   - 迷宫墙体
   - 怪物出生点
   - 宝箱/医疗包/子弹点
   - 逃生车和停车场位置
3. 导出 JSON。
4. 游戏读取 JSON 覆盖默认生成逻辑。

风险点：

- 当前森林是无限程序生成地图，编辑器应先支持“固定关卡区域”，不要一开始编辑无限地图。

## 桌面端打包

已新增 `DESKTOP_PACKAGING.md`，建议优先用 Tauri：

- 包体小。
- 适合当前静态前端结构。
- `localStorage` 存档可继续沿用。
