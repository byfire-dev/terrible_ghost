## 二、核心玩法架构（三阶段循环）

```
┌─────────────────────────────────────────────────────────────┐
│                     第 N 天 游戏循环                          │
├─────────────────────────────────────────────────────────────┤
│  阶段1: 森林探索                                              │
│    ├── WASD移动 + 鼠标瞄准射击                                  │
│    ├── 草丛隐藏（10秒后消失，怪物升2级）                          │
│    ├── 收集子弹/医疗包/宝箱                                     │
│    ├── 击败怪物获得积分 → 商城购买武器/子弹                       │
│    └── 生命归零 → 进入阶段2                                    │
│                          ↓                                   │
│  阶段2: 迷宫挑战                                              │
│    ├── 固定路线迷宫（50×34格，48px/格）                         │
│    ├── 绕开小怪物（阻挡移动）和大怪物                            │
│    ├── 收集2把钥匙 → 开启大门                                   │
│    └── 生命归零 → 回到迷宫起点（丢失钥匙）                       │
│                          ↓                                   │
│  阶段3: 载具逃生                                              │
│    ├── 找到逃生车 → 上车驾驶                                    │
│    ├── 无法射击，只能躲避追击怪物                                │
│    ├── 车辆损坏 → 下车徒步（生命50）                            │
│    └── 到达停车场 → 完成当天 → 进入第N+1天                     │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 渲染管线（每帧）

```
loop() → update(dt) → draw()

update(dt):
  ├── 冷却计时器递减
  ├── 射击处理
  ├── 玩家移动 + 碰撞检测
  ├── 草丛隐藏逻辑
  ├── 子弹更新 + 碰撞
  ├── 怪物AI更新 + 碰撞
  ├── 命中检测
  ├── 拾取物收集
  ├── 粒子更新
  ├── 生成计时器（弹药/宝箱/医疗包/怪物）
  └── HUD更新

draw():
  ├── 清空画布
  ├── 视差震动偏移
  ├── 场景渲染（森林/迷宫）
  ├── 物品渲染（弹药/医疗包/宝箱/钥匙/车门/车/停车场）
  ├── 特效渲染（枪口火焰/子弹/粒子）
  ├── 怪物渲染
  ├── 玩家渲染
  ├── 草丛覆盖层
  ├── 天气效果（雨/雪/雾/灰烬/毒雾/红月/余烬）
  ├── 视野遮罩（手电筒效果）
  ├── 伤害叠加层
  └── 车辆指针DOM更新
```

## 四、关键系统设计详解

### 4.1 武器系统（24级）

| 等级 | 名称 | 伤害 | 射速(发/秒) | 弹速 | 射程 | 价格 | 特性 |
|------|------|------|------------|------|------|------|------|
| 1 | 一级手枪 | 1 | 5.6 | 560 | 62m | 0 | 初始武器 |
| 2 | 二级步枪 | 2 | 7.1 | 640 | 67m | 45 | - |
| 3 | 三级弩枪 | 3 | 5.0 | 700 | 84m | 90 | **可破石** |
| 4 | 四级霰弹枪 | 5 | 3.6 | 590 | 53m | 140 | - |
| 5 | 五级火焰枪 | 7 | 6.3 | 610 | 50m | 230 | - |
| ... | ... | ... | ... | ... | ... | ... | ... |
| 24 | 24级武器 | ~110 | ~10.5 | ~1138 | ~131m | ~17,000 | 循环类型 |

- **10-24级**: 程序化生成，基于9种基础武器类型循环
- **价格公式**: `round(920 + (level-9)*(level-5)*34)` 再乘 `1.85 + level*28`
- **破石机制**: 仅3级及以上武器可破坏岩石（需10发）

### 4.2 敌人AI系统

**三种敌人类型:**

| 类型 | 出现场景 | 速度 | 血量 | 特性 |
|------|---------|------|------|------|
| 普通怪物 | 森林 | 54-82 + danger*5 | 1+level*1.8 | 追踪玩家 |
| 小怪物(small) | 迷宫 | 112 | 1 | **阻挡玩家移动** |
| 大怪物(mazeBig) | 迷宫 | 78+level*3 | 5+level*2.4 | 高伤害 |
| 追击者(chaser) | 载具阶段 | 230-285 | 5+level*2.2 | 高速追击 |

**AI行为逻辑:**
- 森林中玩家躲草丛 → 怪物进入巡逻模式（速度×0.42，随机角度）
- 迷宫中小怪物物理阻挡玩家（碰撞即受伤）
- 怪物碰到岩石 → 反弹并改变方向
- 等级≥3的怪物带拖尾特效

### 4.3 场景系统（12种随机森林）

基于 `sin(day * 91.73)` 伪随机选择，每天不同场景：

| 场景 | 天气 | 特色 |
|------|------|------|
| 雾绿森林 | 雾 | 经典绿色调 |
| 冷月松林 | 雨 | 蓝色冷调 |
| 赤叶荒林 | 余烬 | 橙红色调 |
| 紫雾密林 | 雾 | 紫色调 |
| 灰烬林地 | 灰烬 | 灰色调 |
| 蓝苔湿地 | 雾 | 青色调 |
| 金雾林 | 余烬 | 金色调 |
| 黑杉谷 | 雨 | 深色调 |
| 霜白林 | 雪 | 白色调 |
| 毒沼森林 | 毒雾 | 绿色毒雾 |
| 暗红林 | 红月 | 血红滤镜 |
| 星光林 | 雪 | 蓝色星光 |

### 4.4 视野与氛围系统

- **手电筒效果**: 玩家前方扇形光照（半径250px，角度±0.45rad）
- **边缘黑雾**: 径向渐变遮罩，营造紧张感
- **伤害红屏**: 受击/低血量/怪物靠近时红色叠加层
- **屏幕震动**: 受击时随机偏移（最大13px）

### 4.5 经济系统

| 来源 | 收益 |
|------|------|
| 击败怪物 | level * 5 积分（最低5） |
| 宝箱（55%概率） | 20-95积分 × 天数加成 |
| 宝箱（45%概率） | 随机等级医疗包 |

| 消费项 | 价格 |
|--------|------|
| 武器购买 | 0 ~ ~17,000 积分 |
| 子弹（1-3级） | 每包12发 |
| 子弹（4-24级） | 每包10发 |
| 子弹价格公式 | `round(14 + level*9 + level²*1.35)` |

## 五、代码质量评估

### 5.1 优点

1. **模块化清晰**: ES Modules 拆分合理，职责分离
2. **纯原生实现**: 零依赖，无框架负担
3. **性能优化**: 视锥剔除（只渲染屏幕内对象）、对象池思想
4. **可配置性强**: 大量参数集中在 config.js
5. **视觉效果丰富**: 天气系统、粒子特效、视野遮罩、伤害红屏

### 5.2 潜在问题

| 问题 | 位置 | 严重程度 |
|------|------|---------|
| 全局状态污染 | `main.js` 中 `game` 对象全局可变 | 中 |
| 无类型安全 | 纯JS，无TypeScript | 低 |
| 渲染与逻辑耦合 | `draw()` 和 `update()` 在同一文件 | 中 |
| 魔法数字较多 | 部分数值硬编码 | 低 |
| 无单元测试 | 无测试框架 | 中 |
| 存档系统缺失 | 刷新页面进度丢失 | 高 |
| 音频系统缺失 | 无音效/BGM | 中 |
| 移动端不支持 | 无触摸控制 | 中 |

## 六、改进建议

### 6.1 短期（1-2周）

1. **添加本地存档**: `localStorage` 保存天数/武器/积分
2. **添加音效**: 射击、受击、拾取、背景音乐
3. **移动端适配**: 虚拟摇杆 + 自动射击
4. **Bug修复**: 检查草丛隐藏与医疗包使用的交互逻辑

### 6.2 中期（1个月）

1. **代码重构**: 将渲染系统抽离为独立模块
2. **添加更多场景**: 扩展到20+种场景
3. **Boss战**: 每5天增加一个Boss
4. **成就系统**: 连续天数、无伤通关等

### 6.3 长期（3个月）

1. **多人联机**: WebSocket 合作/对战模式
2. **关卡编辑器**: 自定义迷宫
3. **Steam/Electron 打包**: 桌面端发布
4. **TypeScript 迁移**: 类型安全

## 八、总结

《迷雾森林逃生》是一个设计精巧的浏览器生存射击游戏，核心循环清晰（探索→惩罚→逃生），三阶段玩法提供了丰富的策略选择。代码结构合理，视觉效果出色，尤其适合作为独立游戏原型或教学项目。

**核心亮点**: 草丛隐藏机制、三阶段循环、12种场景变化、24级武器成长
**最需要改进**: 存档系统、音效、移动端支持
