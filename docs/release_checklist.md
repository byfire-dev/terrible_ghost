# 发布验收清单

发布版本：`2026.05.09.5`

## 缓存策略

- `sw.js` 使用版本号缓存：`terrible-ghost-2026-05-09-5`。
- HTML 使用网络优先，避免上线后入口页长期停留在旧版本。
- JS/CSS/WebP 使用缓存优先，提升二次进入和移动端加载速度。
- 每次发布改动代码、样式、资源时，同步更新：
  - `sw.js` 内的 `CACHE_VERSION`
  - `index.html` 内的 `app-version`

## 发布前自动检查

运行：

```bash
npm run release:check
```

需要全部通过：

- JS 语法检查
- TypeScript 检查
- 桌面端启动、进入游戏、Canvas 非空
- 移动端启动、进入游戏、摇杆/射击按钮可见
- Service Worker 注册和缓存激活
- 首屏图片只加载 WebP

## 手动部署步骤

1. 确认 `npm run release:check` 通过。
2. 确认 `assets/images/*.webp` 存在，页面引用不再指向 PNG。
3. 使用静态托管发布整个目录的运行所需文件：
   - `index.html`
   - `game.js`
   - `styles.css`
   - `sw.js`
   - `src/`
   - `assets/images/*.webp`
4. 部署到 HTTPS 域名后，强制刷新一次页面，确认 Service Worker 正常激活。
5. 手机网络下首次打开，确认首页 3 秒内有可见主视觉，点击开始后能进入剧情页。
